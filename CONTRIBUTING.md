# Contributing

## How to propose a change

To contribute, fork this repository and open a pull request against `main`. Describe the change clearly in the PR description and reference any relevant issues. Keep changes focused — one logical change per pull request.

## Style

Follow the existing code style and conventions found in the repository. Keep commits small and well-described. Use conventional commit messages where applicable (e.g., `feat:`, `fix:`, `docs:`).

## Who reviews

Pull requests are reviewed by the maintainers listed in the repository. At least one maintainer approval is required before a PR can be merged. Reviewers may request changes or clarifications before approving.
