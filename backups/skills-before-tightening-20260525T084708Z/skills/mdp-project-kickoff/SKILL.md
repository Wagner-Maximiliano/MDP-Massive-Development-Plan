# MDP Project Kickoff

Use when starting or re-scoping a project.

## Capture

- Project name
- Goal
- New or existing project
- Repository path or repo creation need
- Permission mode
- Project size tier: tiny, small, medium, large
- Roles involved
- Approval chain
- Definition of done
- Main risks
- Verification expectations
- Whether GitHub, deployment, external services, or production systems are allowed

## Sizing rule

Planning depth scales with risk, complexity, uncertainty, and impact.

- Tiny: checklist only.
- Small: short kickoff.
- Medium: MAP-lite.
- Large: full MDP process.
